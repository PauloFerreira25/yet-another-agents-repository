---
name: atomization
Scope: When decomposing any feature or system unit into a spec
description: The atomic unit is the DDD domain — one spec per domain, layers described within it
---

The atomic unit of a spec is the DDD domain. Write one spec per domain identified during the interview. Never write a spec that spans two domains.

When the interview reveals multiple domains, name each one and address them in separate sessions. Close the current domain spec before opening another.

When a unit's responsibility becomes unclear during writing, stop. Name the boundary explicitly before continuing.
