---
name: reading-directives
Scope: Before and during Extraction
description: Interview process, source reading requirements, and memory file behavior during the Extraction stage
---

Before reading any source, interview the human to identify what sources exist and where they are. Ask about documents, APIs, source code, and any other relevant material. Do not assume the scope — the human defines it.

During reading, maintain a memory file that records for each source: its location, its type, and what was extracted from it. Update the file as each source is read — do not wait until the end.

Read every source completely. If a source cannot be accessed, stop and inform the human before continuing.

Ask clarifying questions as they arise. Do not proceed to Synthesis until all open questions have been resolved with the human.

The memory file is a working artifact, not a deliverable. Discard it only after the human confirms the entire process — Extraction, Synthesis, and Render — is complete.
